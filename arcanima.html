Combat System

Arcanima combat system breaks down in two parts: a traditional hack n slash combat part and a focus mode part where the time slows down and a menu is displayed to perform different actions like magical skills, especial attacks or use items.

Real time combat features:

Light and heavy attacks, with the possibility to interject them at any time during the combo.

Blocking, dodging, counter attacks and unblockable attacks.

Input buffering and animation cancellations, essential to achieve a satisfying combat.

Aerial combat.

Singular attacks like launcher, smasher or dash attack.

Automatic and manual lock-on system.

Focus time combat features:

Inventory.

Anima change, players can change their anima anytime while in focus mode, modifying the set of skills they can use

Singular standalone abilities like powerful attacks or magical healing.

Strengths and weaknesses system, some characters' animas are weak or strong to others.



In order to achieve a satisfactory combat, iteration and polish in the attacks logic is of great importance, but at the same time it is necessary that this iteration is done with ease. Movement System

Arcanima movement system presents the following features:

Parameterized movement based on acceleration and frictions, differentiating between grounded and aerial.

Double jump, with jump parameterization based on time or distance.

Displacement subsystem, used on dashing, atacking or being knockbacked. Supports horizontal, vertical and parabolic displacements, all parameterized by distance and time.

Automatic ledge climbing.

Sprinting, running and walking speeds.

Software Engineering in Arcanima

As the technical director of the game I was also responsible for its software design:


Character Architecture: command pattern is used in order to decouple input and behaviour, this way we can have a common interface for characters but different inputs methods. The player character can implement a buffer that stores and reads from user input but an enemy can receive its input from a decision making algorithm such as a behaviour tree. Watch the simplified class diagram here.

This method were used to parallel AI programming and Behaviour programming, making us to work faster and helping us find bugs easier. This architecture also made us able to control enemies and bosses by hand or by a customized string of commands, easing the iteration and polish of their behaviour without the need of an AI operating the character.


Data oriented design all kind of features were build with a data-driven design in mind. Attacks, character stats, displacements, skills and magic abilities, inventory items, camera shake, gamepad rumble... were data objects. The best of this approach is the programming agnostic aspect that able designers and artist to just modify data in order to see their changes reflected.
